# 对话记录摘要 · Poker Battle 兵棋游戏开发全过程

> 本文档记录了从零到一完成 Poker Battle 1.8 卡牌战棋游戏开发的完整过程。

---

## 一、初始版本（自定义简化规则）

1. **核心引擎**：Python 实现 52 张牌兵棋游戏（大本营 + 3 阵地 + 手牌）
2. **命令行 CLI**：双人轮流操作，信息隐藏
3. **网页版 UI**：内置 HTTP 服务器 + 单页 JS 前端
4. **AI 对手**：启发式策略 NPC
5. **单元测试**：覆盖核心规则

## 二、规则迭代与修正

1. 攻击结算改为"两张牌合并计算"（非分别结算）
2. 部署日志记录具体牌名
3. 大本营被迫翻出的牌只算裸点（A=11，不乘 2，不抽牌）
4. 弃牌堆不可见 + 公共区抽空时回收弃牌堆
5. 每次操作实时决策（移除"偏好"系统）
6. 调试用大本营查看按钮

## 三、C++ 自博弈进化系统（v1）

1. C++ header-only 引擎（含策略参数化 + Elo 排名 + Arena）
2. pybind11 绑定（因 MinGW/MSVC 不兼容改用独立 exe）
3. Python 进化调度器（策略变异/交叉/随机 + 淘汰）
4. **百万轮进化**：128 策略池，8 秒完成，发现"潮水"策略碾压

## 四、按原版规则书重写（Poker Battle 1.8）

### 规则差异分析
- 对照 PDF 规则书逐条对比
- 发现 17 处重大差异（待战区、准备阶段、沉默、急救、大本营惩罚等）

### 完整重写
- **models.py**：作战区/待战区/牌库/手牌/墓地 五区域
- **engine.py**：准备阶段、5 种行动、攻击全流程（溢出/击毁/沉默/急救/四象）
- **web.py**：HTTP API + 攻击生成器驱动
- **前端**：左右分栏布局（场地始终可见 + 右侧决策面板）

### 测试覆盖
- 68 个引擎测试 + 15 个 Web API 集成测试 = **83 个测试全通过**
- 覆盖所有基础规则 + 边界情况 + 机制组合

## 五、大规模 AI 策略进化（v2）

### C++ Arena v3
- Poker Battle 1.8 完整引擎 C++ 实现
- 23 维策略参数（操作优先/攻击阈值/状态自适应/A选择/部署/准备/防御）
- 10 局/匹配（5 牌组 × 2 先后手），消除牌组偏差
- 速度：~20,000 匹配/秒

### 10 阶段动态进化
- 每阶段 100 万匹配（= 1000 万局）
- 阶段间分析：参数收敛检测 + Top vs Bottom 差异 + 新策略生成
- 总计 **1000 万匹配 = 1 亿局**，耗时 8.3 分钟

### 冠军策略发现
- Elo 1960，胜率 68.8%
- 核心：低门槛持续攻击 + 100% 全部署 + 频繁征兵 + 梅花集中 + 优先打大本营

## 六、AI 部署到网页 + 公网发布

1. 冠军参数写入 `wargame/ai.py` 作为 `ChampionAI`
2. Web 层支持 vs AI 模式（AI 自动换牌/部署/攻击/防御决策）
3. Cloudflare Tunnel 暴露到公网：`https://xxx.trycloudflare.com`
